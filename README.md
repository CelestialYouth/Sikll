# AI Draw.io 图表生成专家 — Skill 使用说明

> 本说明旨在展示 Skill 学习能力与设计思路，结合 `next-ai-draw-io` 项目，将 AI 驱动的图表生成能力封装为可复用的 TRAE Skill。

---

## 一、项目本质理解

`next-ai-draw-io` 是一个 **AI 驱动的图表创建工具**，其核心能力包括：

- **自然语言生成/修改 draw.io 图表**：用户用日常语言描述需求，AI 自动输出 draw.io 兼容的 XML。
- **多 AI 提供商支持**：OpenAI、Claude、DeepSeek、Gemini 等均可接入。
- **MCP Server 供 AI Agent 调用**：项目已提供 `@next-ai-drawio/mcp-server`，可被 Claude Code 等 Agent 直接调用。
- **XML 格式存储与渲染**：图表以标准 draw.io XML 格式输出，可直接在 draw.io 或嵌入网页中打开。

本项目本身即是一个 **"AI 工具"**，将其封装为 Skill 的关键切入点如下：

## 三、Skill 类型确定

根据 Anthropic 的 9 类 Skill 分类，本 Skill 属于以下类型：

## 四、关键知识提取

AI 需要掌握但无法自行推断的核心知识：

### 4.1 draw.io XML 格式规范

- 根节点为 `<mxfile>`，内部包含 `<diagram>`，再嵌套 `<mxGraphModel>`。
- 所有图形元素均为 `<mxCell>`，通过 `vertex="1"`（节点）或 `edge="1"`（连线）区分。
- 几何信息通过 `<mxGeometry>` 子节点定义。

### 4.2 不同图表类型的最佳实践

| 图表类型 | 适用场景 | 关键要点 |
|----------|----------|----------|
| 流程图 | 业务流程、算法流程 | 从上到下，判断用菱形，开始结束用椭圆 |
| 架构图 | 系统架构、微服务架构 | 按层分组，统一颜色方案，标明边界 |
| 时序图 | 系统交互、API 调用链 | 明确参与者生命线，消息流向清晰 |
| 云架构图 | AWS/GCP/Azure 部署 | 使用官方 `resIcon` 图标，按服务分层 |

### 4.3 云架构图的特殊要求

- **Claude 系列** 在云架构图（AWS/GCP/Azure）上表现最佳，训练数据包含大量带云厂商 Logo 的 draw.io 图表。
- 使用 `resIcon=mxgraph.aws4.xxx` 引用官方图标，确保视觉专业度。
- 按网络层、应用层、数据层垂直排列，添加 VPC/子网边界框。

### 4.4 模型选择建议

| 图表类型 | 推荐模型 | 原因 |
|----------|----------|------|
| 云架构图 | **Claude 系列** | 训练数据丰富，AWS/GCP/Azure 图标识别准确 |
| 通用流程/架构 | GPT-4.5/GPT-5、DeepSeek V3.2/R1 | 通用能力强，性价比高 |
| 需结合图片参考 | Gemini 3 Pro | 多模态能力突出，可基于参考图生成 |

## 五、Skill 结构设计

### 5.1 触发条件

当用户输入包含以下关键词时自动激活：

> 画图、架构图、流程图、时序图、系统图、画个图、生成图表、draw.io、diagram、flowchart、architecture diagram、画架构、画流程

### 5.2 核心指令

指导 AI 如何构造有效的 draw.io XML：

1. **理解需求**：分析用户想要表达的系统/流程/架构。
2. **选择图表类型**：根据内容选择最合适的图表类型。
3. **生成 XML**：构造符合 draw.io 格式的 XML，确保标签闭合、ID 唯一。
4. **优化布局**：遵循从上到下、从左到右原则，确保间距足够、对齐网格。
5. **输出结果**：提供可直接保存为 `.drawio` 或嵌入网页的 XML。

### 5.3 坑点清单（Gotchas）

| 坑点 | 说明 | 解决方案 |
|------|------|----------|
| XML 格式严格 | 缺少闭合标签或错误属性会导致无法渲染 | 生成后自查标签闭合与特殊字符转义 |
| 坐标重叠 | 元素坐标重叠会导致视觉混乱 | 同级间距至少 40px，层级间距至少 60px |
| 连接点错误 | `exitX/exitY` 和 `entryX/entryY` 设置错误 | 根据元素相对位置设置正确连接点 |
| ID 重复 | `mxCell` ID 必须唯一 | 使用有意义的命名，如 `user-service`、`auth-db` |
| 模型能力差异 | 不是所有模型都能生成复杂图表 XML | 云架构图优先用 Claude，通用图用 GPT/DeepSeek |
| 长文本溢出 | 节点标签过长不会自动换行 | 设置 `whiteSpace=wrap` 并控制宽度 |
| 颜色对比度 | 相近颜色导致可读性差 | 使用预设颜色方案，确保文字与背景对比度足够 |

### 5.4 脚本/模板

提供常用图表类型的 XML 模板，加速生成：

- **基础结构模板**：`<mxfile>` → `<diagram>` → `<mxGraphModel>` → `<root>` 的标准骨架。
- **元素模板**：矩形（进程）、菱形（判断）、椭圆（开始/结束）、连接线、AWS 图标等。
- **颜色方案模板**：用户/外部、前端/客户端、后端服务、数据库、缓存/CDN、消息队列等预设配色。

## 七、MCP Server 使用（推荐）

若用户环境已配置 MCP，优先使用官方 MCP Server：

```bash
# Claude Code CLI 安装
claude mcp add drawio -- npx @next-ai-drawio/mcp-server@latest
```

配置后可直接通过自然语言调用：

- "创建一个展示微服务架构的图"
- "画一个用户登录认证的流程图"
- "生成一个 AWS 三-tier 架构图"

## 九、进阶技巧

- **动画连接器**：在 style 中添加 `animated=1`，突出关键数据流。
- **分组容器**：使用 `swimlane` 或 `group` 样式组织相关元素，如 VPC、子网。
- **多页图表**：对于复杂系统，使用多个 `<diagram>` 标签分页展示概览与详情。

## 十、故障排查速查表

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 图表无法打开 | XML 格式错误 | 检查标签闭合、特殊字符转义 |
| 元素不显示 | 坐标超出视口 | 调整 x/y 坐标到合理范围 |
| 连线不连接 | source/target ID 错误 | 确认 ID 与目标元素匹配 |
| 样式不生效 | 样式字符串格式错误 | 检查分号和等号使用 |
| 图标不显示 | resIcon 值错误 | 使用正确的图标标识符 |
